# Input

An input field coupled with a label and supplemental helper text that aids in user input and feedback.

## Guidelines

### When to use

- To gather basic usage information.
- For responses that are a single line of text.

### When not to use

- For responses longer than one line. Consider using a [Textarea](/docs/components-textarea--docs).
- For responses from a predefined list. Consider using a [Select](/docs/components-select--docs).

### Accessibility

For accessibility, it is strongly encouraged to follow these best practices:

- Use the `label` property to help screen reader users identify the purpose of the input.
- Do not use a `placeholder` to communicate the field label, use a `label` property.
- Provide clear and descriptive error, helper, and placeholder messages.

### Default

The type of input used varies based on the type of information you look to capture. The `text` default type allows for any type of single line data. For more information on the types this component supports, please see the MDN article on the [input element](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input#input_types).
