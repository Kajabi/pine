# Radio Group

Radio Group components wrap multiple `pds-radio` components and display a single error message for the entire group, providing a cleaner interface for grouped radio selections.

## Guidelines

### When to use
- When you have multiple radio options that belong to the same selection group
- When you want a single error message displayed for the entire group instead of individual error messages
- For form validation where the group must have a selection

### When not to use
- For single radio buttons; use `pds-radio` directly
- When each radio needs its own individual error message
- For checkboxes; use `pds-checkbox` components instead

### Accessibility
- Ensure the group label is descriptive and clearly identifies the selection group
- Use proper error messaging and states (e.g., `invalid`) for validation feedback
- The `name` attribute is automatically applied to all child radios to ensure proper grouping

## Form Integration

- The `name` prop is automatically applied to all child radios, ensuring proper form grouping
- Set `required` on the group to mark all radios as required for form validation
- Use `invalid` and `errorMessage` to display validation feedback when no choice is selected
- Each radio should have a unique `componentId` and `value` for proper form submission
