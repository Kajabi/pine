# sage-button

A button component to generate all versions of `sage-button` as outlined in [Figma](https://www.figma.com/file/sMbtLUHSt2vfKgKjyQ3052/%F0%9F%A7%A9-Sage-components?node-id=3%3A15&t=dIcvX1VtOu0FL0VE-0).

## Usage Guidelines

### When to use

- Communicating an action that will occur. (e.g. saving a page)
- Triggering or enabling an action, such as submitting requested information. (e.g. forms)
- Progressing or regressing a user through a step in a flow. (e.g. modals)

### When not to use

- Directing users to a new page or different part within the same page. Instead, use [sage-link](/docs/webcomponents-sage-link--external).
- Limited space available. Instead consider an IconButton.

## Type

The type prop allows a `sage-button` to set [button type attribute](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/button#attr-type).

### Default (Button)

The button has no default behavior, and does nothing when pressed by default. It can have client-side scripts listen to the element's events, which are triggered when the events occur.

This is our default button type.

### Submit

The button submits the form data to the server.

### Reset

The button resets all the controls to their initial values. [As noted on MDN, this behavior tends to annoy users](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/button#attr-type).
