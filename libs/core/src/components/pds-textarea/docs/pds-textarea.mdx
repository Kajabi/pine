# Textarea

A textarea field coupled with a label, helper message, and error message which allows for user input and feedback

## Guidelines
### Usage

#### When to use
- You want a response longer than one line

#### When not to use
- To gather basic information, use an [Input](/docs/components-input--text)
- You want a response that is one line of text, use an [Input](/docs/components-input--text)

### Accessibility
- The textarea component comes with a built-in label through the `label` property, it is strongly encouraged that you include it
- Please provide clear and descriptive error, helper-message, and placeholder messages

### Error Message

Error messages are passed using the `error-message` property, textareas will only calculate their validity when on their `required` property is set to true

You can also manually pass in an invalid state by setting the `invalid` property to true but this property will be update whenever the `onChange` event is triggered.

<DocCanvas client:only
  mdxSource={{
    react: '<PdsTextarea name="Invalid" label="Name" errorMessage="Error" invalid="true" required="true" componentId="textarea-invalid"></PdsTextarea>',
    webComponent: '<pds-textarea name="Invalid" label="Name" error-message="Error" invalid="true" required="true" component-id="textarea-invalid"></pds-textarea>'
}}>
  <pds-textarea name="Invalid" label="Name" error-message="Error" invalid="true" required="true" component-id="textarea-invalid"></pds-textarea>
</DocCanvas>

### Helper Message

Helper messages are used to provide context about what is expected from the users

Do not use helper messages to provide examples to users, please use the `placeholder` property instead

### Placeholder

Placeholders should be examples of content that can be filled into the textarea

Do not use placeholders to provide hints to users, please use the `helper-message` property instead
