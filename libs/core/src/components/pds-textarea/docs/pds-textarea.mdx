# Textarea

A textarea field coupled with a label and supplemental helper text that aids in user input and feedback.

## Guidelines

### When to use

- For responses that require text longer than a single line.

### When not to use

- For responses that require basic information or a single line of text. Consider using an [Input](/docs/components-input--docs).

### Accessibility

For accessibility, it is strongly encouraged to follow these best practices:

- Use the label property to help screen reader users identify the purpose of the input.
- Do not use a placeholder to communicate the field label, use a label property.
- Provide clear and descriptive error, helper, and placeholder messages.

### Error Message

Error messages are passed using the `error-message` property. Textarea fields will only calculate their validity when their `required` property is set to true.

The `invalid` property can be manually set to `true` to enable the invalid state. This property will be updated when the `onChange` event is emitted.

<DocCanvas client:only
  mdxSource={{
    react: '<PdsTextarea name="Invalid" label="Name" errorMessage="Field cannot be blank" invalid="true" required="true" componentId="textarea-invalid"></PdsTextarea>',
    webComponent: '<pds-textarea name="Invalid" label="Name" error-message="Field cannot be blank" invalid="true" required="true" component-id="textarea-invalid"></pds-textarea>'
}}>
  <pds-textarea name="Invalid" label="Name" error-message="Field cannot be blank" invalid="true" required="true" component-id="textarea-invalid"></pds-textarea>
</DocCanvas>

### Helper Message

A `helper-message` is used to provide additional information about how to complete the text field.

Do not use helper messages to provide examples to users, please use the `placeholder` property instead.

}}>
  <pds-textarea name="Message" label="Name" helper-message="Helper message text" component-id="pds-textarea-helper-example"></pds-textarea>
</DocCanvas>

Removes the ability to interact with the text field. If a value is present, that is still passed to the form.

Indicates that the user must specify a value for the input before the owning form can be submitted. It's recommended to use `error-message` on any required fields.

### Action Slot

The action slot allows you to add helpful content next to the textarea label, such as links or help buttons. This is useful for providing additional context or quick actions related to the field.

**Action Link**

<DocCanvas client:only
  mdxSource={{
    react: `<PdsTextarea componentId="pds-textarea-action-link" label="Notes" rows="3" helperMessage="Add any additional notes here">
  <PdsLink href="#" slot="action">View examples</PdsLink>
</PdsTextarea>`,
    webComponent: `<pds-textarea component-id="pds-textarea-action-link" label="Notes" rows="3" helper-message="Add any additional notes here">
  <pds-link href="#" slot="action">View examples</pds-link>
</pds-textarea>`
}}>
