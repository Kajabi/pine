# Select

The Select component provides a dropdown list of options, allowing users to make a selection from a predefined set.

## Usage Guidelines

### When to use
- When a user needs to select options from a predefined list.
- For forms with multiple dropdowns requiring consistent styling and behavior.
- When displaying grouped or categorized options for easier navigation.

### When not to use
- For binary choices; consider a radio or switch instead.
- When users need to input free-form text; a text field may be better suited.
- For very long lists of options where search or filtering would be more effective.

### Accessibility
- Use meaningful labels and associate them clearly with the select element.
- Provide helper text or error messages for guidance and validation.
- Avoid overloading dropdowns with excessive options to prevent overwhelm.

### With Optgroup

The `optgroup` element allows for grouping related options within the select component, making it easier for users to navigate large lists by categorizing options. This is especially useful when the list contains a variety of similar items, such as countries grouped by continents or product categories grouped by type.

<DocCanvas

  mdxSource={{

      <PdsSelect componentId="pds-select-optgroup-example" label="Name" value="george">

          <option value="george">George Harrison</option>
