# Popover

Popovers display contextual information when users click or fires a keyboard event on an element. They typically contain more information than tooltips and can include interactive elements.

## Guidelines
### Usage
- Use popovers for complex information that might overwhelm the main content.
- Consider using a tooltip for simpler explanations or brief labels.

#### When not to use
- Don't use popovers for critical information that must be readily available.

## Variants
### Default
The popover's appearance and behavior are primarily controlled by two slots:
1. **`trigger`:** This slot is used to define the element that triggers the popover. It can be any HTML element, such as a button or link.
2. **`content`:** This slot is used to define the content of the popover. It can contain any HTML elements, including text, images, forms, or other components.

### Positioning
Using the placement prop, you can control where the popover appears relative to the trigger.

### Arrow
The popover includes an arrow by default, but you can hide it with the `has-arrow` prop.

### Width/Sizing
The `maxWidth` prop allows for adjust the maximum width for the tooltip content. The default max width is 352px, but it can be customized as needed by passing a new pixel value.
