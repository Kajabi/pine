# Button

A button is an interactive element that can be clicked to perform various actions such as triggering an event, submitting data, or navigating to a different location.

## Guidelines

### When to use

- Communicating an action that will occur (e.g. saving a page).
- Triggering or enabling an action, such as submitting requested information (e.g. forms).
- Progressing or regressing a user through a step in a flow (e.g. modals).

### When not to use
- When space is limited consider an IconButton.

### Accessibility

While accessible out of the box, there are some best practices to keep in mind when using the component:

- Use helpful text to describe the button's action even if this text is expected to be visually hidden (e.g. "submit form" or "delete account"). Visually hidden text will still be read by screen readers.
- Buttons should not disable tabbing or keyboard navigation.
- Disabled buttons will not be read by screen readers.

### Buttons vs Links

In many current web design patterns, buttons and links have evolved to look visually identical. However, for best practices, the fundamental differences between the two should be considered before using. Links are used to navigate the user to another page. Buttons are used to perform an action on the current page or submit a form.

### Icon Slots

The `start` and `end` slots allow icons to be placed before and after the button text but will be hidden when `loading` is true. For more information on web component slots, see the [MDN documentation](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_templates_and_slots#adding_flexibility_with_slots).

```html
<pds-button>
  <pds-icon slot="start" aria-hidden="true" name="trash"></pds-icon>
  Start Icon
</pds-button>
```

```html
<pds-button>
  End Icon
  <pds-icon slot="end" aria-hidden="true" name="trash"></pds-icon>
</pds-button>
```

### Sizes

Buttons come in three sizes: default, small, and micro. The size prop allows you to control the button's height and spacing.

- **Default**: Standard button size - used when no size prop is specified
- **Small**: Smaller button size
- **Micro**: Smallest button size with reduced font size

<DocCanvas client:only

### Button Link

The button link variant allows a `pds-button` to function as a link. By providing an `href` attribute, the button will render as an anchor (`<a>`) element while maintaining the visual appearance of a button. This is useful for creating call-to-action buttons that navigate to external resources or different parts of your application.

> ***Note:*** When using the `href` attribute, button-specific props such as `type`, `name`, and `value` will be ignored. The component will maintain the same visual appearance but function as a link.

<DocCanvas client:only
  mdxSource={{
    react: `
    <PdsButton variant="primary" href="#">Create a product</PdsButton>
    <PdsButton variant="secondary" href="#">Join community</PdsButton>
    `,
    webComponent: `
    <pds-button variant="primary" href="#">Create a product</pds-button>
    <pds-button variant="secondary" href="#">Join community</pds-button>
    `
}}>
  <pds-button variant="primary" href="#">Create a product</pds-button>
  <pds-button variant="secondary" href="#">Join community</pds-button>
</DocCanvas>

### Type

The type prop allows a `pds-button` to set a [button type attribute](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/button#attr-type).

> ***Note:*** The `type` prop does not apply to buttons with an `href` attribute. When an `href` is provided, the button will render as an anchor (`<a>`) element, and the types will be ignored.

#### Default (Button)

The button has no default click behavior. It can have client-side scripts listen to the element's events, which are triggered when the events occur.

#### Submit

The button submits the form data to the server.

#### Reset

The button resets all the controls to their initial values. [As noted on MDN](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/button#attr-type), this behavior tends to annoy users.
