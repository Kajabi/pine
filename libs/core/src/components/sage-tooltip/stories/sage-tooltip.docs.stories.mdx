## Variants
### Trigger
Whether your trigger is text or an HTML element, it will be placed in the default slot between the opening and closing tags.

<Story story={stories.Default} />

<Canvas>
  <sage-tooltip content="this is tooltip content" has-arrow="true" placement="bottom-start">
    text
  </sage-tooltip>
  <sage-tooltip content="this is tooltip content" has-arrow="true" placement="right-start">
    <sage-button variant="secondary">Button</sage-button>
  </sage-tooltip>
</Canvas>

### Content
#### Text Content
Text-based content are generated using the `content` prop. Use this option only when the
tooltip content is expected to be a string.

<Canvas>
  <sage-tooltip content="this is tooltip content" has-arrow="true" placement="right">text</sage-tooltip>
</Canvas>

#### HTML Element Content
Tooptip content may contain HTML elements. In these cases,
enable the `htmlContent` property and provide your element in the `content` slot.

<Story story={stories.Slotted} />

<Canvas>
  <sage-tooltip content="this is tooltip content" has-arrow="true" placement="right">
    text
    <div slot="content">
      <p>this is s a sentence in a tooltip.</p>
      <p>this is s a sentence in a tooltip.</p>
    </div>
  </sage-tooltip>
</Canvas>

### Positioning
Using the placement prop you can adjust where your tooltip will be displayed.

<Story story={stories.Positioning} />

### `has-arrow`
By default the arrow is visible but it can be hidden by disabling it when `has-arrow` is `true`.

<Story story={stories.NoArrow} />

<Canvas>
  <sage-tooltip content="this is tooltip content" has-arrow="false" placement="right">text</sage-tooltip>
</Canvas>
