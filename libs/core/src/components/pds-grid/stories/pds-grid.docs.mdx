## Additional Resources

Responsive
Currently have to put size="12" on the column to trigger the responsive, but not sure if I like that


Start / End
Simiar to offset, use start to denote the beginning of a column. It is meant to be paired with end to denote the end of a column. A start with no end will be size to one column

End
Think of end when spanning to the end of the grid. It is meant to be paired with start to denote the end of a column. An end with no start will be size to one column
Setting an end only causes the grid item to be one column length, that may not be correct

Grid CSS Variables
grid columns - Allows for the familiar Bootstrap feel. The bad is that it's style based so it makes maintaining responsive changes more difficult
grid-gap - A way to set the gap between grid items for all breakpoints via a style attribute

Alignment
When alignment is set on the grid the border gets applied to the content. This is not desireable.
This leads me to believe the Box or Flex should be the immediate child and THAT should contain the alignment

Wrapping
If there isn't enough availabe space for the grid item, it will wrap to the next line. This is the default behavior of the grid. To prevent wrapping, use nowrap.
Any gaps used will only apply horizontal and vertical spacing between grid items. 
Note - When there are no sizes on the grid items, , each grid item will automatically be sized to one column with a static width.

Nesting
Nesting allows to place a grid inside of a grid. This is useful for creating more complex layouts.
