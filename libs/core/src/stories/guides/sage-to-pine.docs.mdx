# Sage → Pine migration

Map common Sage-era and raw HTML/CSS habits from the Kajabi monolith onto their Pine equivalents. Most violations in admin UI are not "wrong Pine" — they are Sage habits (div stacks, utility classes, raw headings, max-width wrappers) that were never translated. Use this when converting existing markup to Pine, or when an AI assistant is editing a file that still contains Sage/HTML layout.

> **pine-mcp note:** `get_pine_design_doc` currently runs pulled `.docs.mdx` through an MDX parser that moves fenced code blocks into a separate "Examples" section, which breaks inline before/after pairing. This guide uses **comparison tables** (`pds-table`) instead of fenced blocks so before/after stay together. If pine-mcp preserves fenced code for `guides/` docs, fenced examples can be added later.

## The mental model shift

Pine is **component-first**, not utility-first. Two rules cover most migrations:

1. **Layout is `container → row → box`** — not divs with CSS.
   - `pds-container` caps content to a readable max-width (replaces max-width wrapper divs).
   - `pds-row` + `pds-box size-*` form the responsive 12-column grid (replaces Bootstrap-style `row`/`col-*` and flex utility stacks).
   - `pds-box` is also the primitive for spacing and standalone flex containers.
2. **Content is semantic components** — not styled divs.
   - Text → `pds-text` with a `tag`. Buttons → `pds-button`. Inputs → `pds-input`/`pds-select`/`pds-textarea`. Icons → `pds-icon`. Feedback → `pds-alert`/`pds-toast`.

Spacing comes from props (`gap`, `padding`), not margin utilities or inline styles. Colors come from tokens, not hex.

## Gotchas when migrating

- **Don't reintroduce `style="…"`** — there is almost always a Pine prop (`padding`, `gap`, `display`, `background-color`, `width`). Inline styles are a blocking violation.
- **Don't keep utility classes** — `mt-3`, `col-md-6`, `d-flex` have no effect on Pine web components. Translate them to props.
- **Don't wrap content in a bare `<div>`** inside a Pine layout — use `pds-box`. Raw HTML mixed with Pine components is blocked.
- **Mobile-first sizing** — inside `pds-row`, give each `pds-box` a `size-xs` baseline before larger breakpoints; never use the base `size` attribute.
- **`pds-container` ≠ grid** — it caps width; nest a `pds-row` inside it when you also need columns.
- **Colors are tokens** — replace hex values with semantic tokens (`get_pine_tokens`).
